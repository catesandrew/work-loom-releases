Renderer Copilot Package

The @workloom/renderer-copilot package implements the ProviderRenderer interface for GitHub Copilot. It renders workspace assets into the .github/copilot-instructions.md file and .github/instructions/ directory that Copilot reads at runtime.

Usage​

import { CopilotRenderer } from '@workloom/renderer-copilot'
import { RendererRegistry } from '@workloom/renderer-core'

const renderer = new CopilotRenderer()

// Register with the renderer registry
const registry = new RendererRegistry()
registry.register(renderer)

// Render assets
const files = renderer.renderAssets(assets, {
  workspaceId: 'ws_abc123',
  workspaceName: 'my-workspace',
  packageName: '@workloom/standard-skills',
  packageVersion: '1.0.0',
  targetPath: '.',
})

Supported Asset Types​

Asset TypeOutput Path
Instruction.github/copilot-instructions.md (combined)
Skill.github/instructions/<slug>.instructions.md
Agent.github/instructions/<slug>.instructions.md

Render Plan​

The plan() method produces a deterministic RenderPlan with trust classification:

Trust CategoryBehavior
CommittedProjectionAll files require commit to repository (Copilot reads from committed files only)

All Copilot output requires approval because it must be committed to the repository to take effect.

Output Structure​

.
└── .github/
    ├── copilot-instructions.md            # All instruction assets combined
    └── instructions/
        ├── <skill-slug>.instructions.md   # Per-skill files
        └── <agent-slug>.instructions.md   # Per-agent files

copilot-instructions.md Format​

All instruction assets are combined into a single file with section headings:

<!-- Managed by Workloom — DO NOT EDIT -->
<!-- workspace: my-workspace -->
<!-- package:   @workloom/standard-skills -->
<!-- version:   1.0.0 -->

Per-File Instructions​

Skills and agents are rendered as individual .instructions.md files in .github/instructions/:

<!-- Managed by Workloom — DO NOT EDIT -->
<!-- workspace: my-workspace -->
<!-- ... -->

File names are derived from the asset ID, slugified to lowercase alphanumeric with hyphens.
