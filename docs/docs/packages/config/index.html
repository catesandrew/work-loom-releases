Config Package

The @workloom/config package provides dotenv loading with variable expansion, environment normalization, and a CLI binary (wl-env) for injecting env files into subprocess commands.

Usage​

import { correctEnv, envs, parse, parseJSON } from '@workloom/config'

// Normalize NODE_ENV, ENV, and BROWSERSLIST_ENV
correctEnv({ nodeEnv: 'production' })

// Load and merge .env files for the current environment
const resolved = envs({ appDirectory: '/path/to/app' })

wl-env CLI​

A binary for loading an env file and running a command with those variables injected into process.env.

wl-env -e .env.local -- node server.js

Flags:

FlagDescription
-e <file>Path to the env file to load (supports dotenv and JSON formats)
--Separator between wl-env flags and the command to run

The loaded variables are expanded (variable interpolation via dotenv-expand) and injected into the environment before spawning the child process.

Key Exports​

correctEnv(options?)​

Normalizes environment variables by setting NODE_ENV, ENV, and BROWSERSLIST_ENV on process.env. Reads from existing process.env values first, falls back to provided options.

correctEnv({ nodeEnv: 'development', env: 'staging', browserslistEnv: 'development' })

envs(options?)​

Loads and merges a cascade of .env files based on the current NODE_ENV and ENV values:

  1. .env
  2. .env.local (skipped when NODE_ENV === 'test')
  3. .env.{ENV} (e.g. .env.staging)
  4. secretsPath (custom secrets file)
  5. .env.{ENV}.local
const env = envs({
  appDirectory: process.cwd(),  // Base directory for resolving .env files
  secretsPath: '/run/secrets/app.env',  // Optional extra secrets file
  override: false,  // If true, loaded values overwrite existing process.env
})

parse(src, override?, processEnv?)​

Parses a dotenv-formatted string with variable expansion. When override is true, parsed values are written to process.env.

const vars = parse('API_URL=http://localhost:${API_PORT}')

parseJSON(src, override?, processEnv?)​

Same as parse, but expects the source string to be JSON (an object of key-value pairs). Variables are expanded after parsing.

const vars = parseJSON('{"DB_URL": "postgres://${DB_HOST}:5432/app"}')
