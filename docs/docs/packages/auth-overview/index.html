Auth Package Overview

The @workloom/auth package handles authentication and authorization for Workloom. It provides OIDC token validation, device authorization flow, role resolution, and the role-provider plugin system.

Capabilities​

ModulePurpose
Device AuthRFC 8628 device authorization flow for CLI login
OIDC ValidatorJWT signature verification via JWKS discovery
Token StoreSecure credential persistence (0600 permissions)
Role ResolutionPlugin-based role determination (see Role Providers)
Role MappingExternal-to-internal role translation with glob patterns

Authentication Flow​

API Server (Fastify)​

Client Request
     │
     ▼ Bearer token
┌─────────────────────┐
│ validateOIDCToken()  │  ← Verifies JWT signature against JWKS
│ or validateDevToken()│  ← Dev mode: validates local dev JWT
└──────────┬──────────┘
           │ ValidatedIdentity
           ▼
┌─────────────────────┐
│  resolveUserRole()   │  ← Runs all providers, maps external roles
└──────────┬──────────┘
           │ WorkloomRole
           ▼
     request.user = { sub, email, org_id, role }

Web App (Next.js)​

Browser Session
     │
     ▼ NextAuth session + access token
┌─────────────────────┐
│ Check role cache     │  ← unstorage (filesystem or S3)
│ (5-minute TTL)       │
└──────────┬──────────┘
           │ cache miss
           ▼
┌─────────────────────┐
│ Fetch /oidc/v1/     │  ← Query IdP userinfo endpoint
│      userinfo        │
└──────────┬──────────┘
           │ claims
           ▼
┌─────────────────────┐
│ mapExternalRoles()   │  ← Shared mapping engine
└──────────┬──────────┘
           │ WorkloomRole
           ▼
     Cache result, return to UI

Dev Mode​

When AUTH_ISSUER is not set, the server runs in dev mode:

  • Tokens are issued/validated locally via issueDevToken() / validateDevToken()
  • No external IdP is required
  • Default role resolution applies (typically viewer unless role mapping config is set)

Role Caching (Web)​

The web app caches resolved roles using unstorage:

  • Local dev: Filesystem at .unstorage/roles/
  • Production: S3 bucket (configured via ROLE_STORAGE_BUCKET + ROLE_STORAGE_REGION)
  • TTL: 5 minutes (re-fetches from IdP on expiry)

Exports​

// Core auth
import { AuthClient, TokenStore, validateOIDCToken, issueDevToken } from '@workloom/auth'

// Role resolution
import { resolveUserRole, RoleProviderRegistry, loadMappingConfig, mapExternalRoles } from '@workloom/auth'

// Providers
import { OIDCRoleProvider, DatabaseRoleProvider, LDAPRoleProvider } from '@workloom/auth'

// Types
import type { WorkloomRole, RoleProvider, RoleMappingConfig } from '@workloom/auth'

// Client-safe (no Node.js deps)
import { ROLE_HIERARCHY, type WorkloomRole } from '@workloom/auth/roles'
