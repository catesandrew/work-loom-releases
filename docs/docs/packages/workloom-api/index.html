Workloom API Client Package

The @workloom/workloom-api package contains auto-generated TypeScript types, React Query hooks, client functions, and Zod schemas for the Workloom REST API. It is generated from the API server's OpenAPI spec using Kubb v4.

Usage​

// Generated React Query hooks
import { useGetPackages, usePostPackages } from '@workloom/workloom-api/hooks'

// Generated client functions (for non-React contexts)
import { getPackages, postPackages } from '@workloom/workloom-api/client'

// Generated TypeScript types
import type { Package, PackageVersion, Workspace } from '@workloom/workloom-api/types'

// Generated Zod schemas
import { packageSchema, workspaceSchema } from '@workloom/workloom-api/zod'

Generated Hooks Pattern​

Every hook follows a consistent pattern that integrates with @workloom/fetch-client:

import { useGetPackages } from '@workloom/workloom-api/hooks'
import { useAccessToken } from '@/hooks/use-access-token'

function PackageList() {
  const { accessToken, headers } = useAccessToken()
  const enabled = accessToken !== null

  const { data, isLoading } = useGetPackages(
    { client: { headers }, query: { enabled } },
  )

  if (isLoading) return <Spinner />
  return <ul>{data?.map(pkg => <li key={pkg.name}>{pkg.name}</li>)}</ul>
}

Hook Options​

All generated hooks accept an options object with:

FieldTypePurpose
client.headersRecord<string, string>Headers passed to the fetch client (typically auth)
query.enabledbooleanPrevents queries from firing before auth is ready
query.*React Query optionsAny standard useQuery / useMutation options

Subpath Exports​

Export PathContents
@workloom/workloom-apiBarrel re-export of all subpaths
@workloom/workloom-api/clientTyped client functions (one per endpoint)
@workloom/workloom-api/hooksReact Query hooks (queries for GET, mutations for POST/PUT/PATCH/DELETE)
@workloom/workloom-api/typesTypeScript interfaces for request/response bodies
@workloom/workloom-api/zodZod schemas for runtime validation

Regenerating​

The generated code lives in packages/workloom-api/src/generated/ and should never be manually edited.

# Regenerate from the API server's OpenAPI spec
pnpm --filter @workloom/workloom-api generate

# Full pipeline: export spec from API, then generate
pnpm --filter @workloom/workloom-api generate:full

Prerequisites​

The API server must have its OpenAPI spec exported to packages/workloom-api/swagger.json:

# Export the spec (starts the API, calls /docs/json, writes swagger.json)
pnpm --filter @workloom/api generate:spec

Kubb Configuration​

The generation is configured in packages/workloom-api/kubb.config.ts:

  • pluginOas — validates the OpenAPI spec
  • pluginTs — generates TypeScript types with asConst enums
  • pluginClient — generates client functions importing from @workloom/fetch-client
  • pluginReactQuery — generates hooks importing from @workloom/react-query
  • pluginZod — generates typed Zod schemas

API Coverage​

Generated hooks and client functions cover all API endpoints:

  • GET/POST /packages — list/create packages
  • GET /packages/:name — package details
  • GET/POST /packages/:name/versions — version management
  • GET/PUT /workspaces — workspace CRUD
  • GET /workspaces/:id/manifest — resolved workspace manifest
  • GET /telemetry/syncs|adoption|providers|drift|failures — telemetry data
  • POST /auth/device|token|refresh — device auth flow
  • GET /auth/whoami — identity check

Dependencies​

  • @workloom/fetch-client — HTTP transport
  • @workloom/react-query — React Query hooks base
  • zod — runtime schema validation
