Drift Package

The @workloom/drift package detects and reports configuration drift between the desired workspace state and the actual local filesystem. It powers the wl status and wl sync --check commands.

Usage​

import { DriftDetector, DriftReporter } from '@workloom/drift'

const detector = new DriftDetector()
const reporter = new DriftReporter()

// Detect drift
const entries = await detector.detect(renderPlans, currentState, workspaceRoot, remoteRevision)

// Format for display
const output = reporter.format(entries, 'human')
console.log(output)

// Use as CI exit code
process.exitCode = reporter.exitCode(entries)

DriftDetector​

Compares render plans against the local filesystem and workspace state to identify drift.

detect(renderPlans, currentState, workspaceRoot, remoteRevision?)​

ParameterTypeDescription
renderPlansRenderPlan[]Expected rendered output from all providers
currentStateStateFile | nullLocal .loom/state.json (null if first sync)
workspaceRootstringAbsolute path to the workspace root
remoteRevisionnumber?Latest revision from the server (optional)

Returns DriftEntry[] — an array of detected drift entries.

Detection Logic​

  1. Remote revision check — compares local state revision against server revision
  2. Content hash comparison — reads each expected file and compares SHA-256 hash against the render plan
  3. Missing file detection — expected files that don't exist on disk
  4. Orphan detection — files tracked in state that are no longer in any render plan

isClean(entries)​

Returns true if no drift was detected.

if (detector.isClean(entries)) {
  console.log('Workspace is in sync')
}

DriftReporter​

Formats drift entries for display or machine consumption.

format(entries, mode)​

ModeOutput
'human'Grouped, labeled output suitable for terminal display
'json'JSON array of drift entries for CI/tooling

Human output example:

Drift detected:

  Remote update available:
    Workspace revision behind: local 41, remote 42

  Modified:
    .claude/skills/security-review/SKILL.md

  Missing:
    .codex/config.toml

exitCode(entries)​

Returns 0 if clean, 1 if any drift detected. Designed for CI pipelines:

wl sync --check || exit 1

Drift Types​

TypeTrigger
LocalModificationFile content hash differs from expected
MissingFileExpected file not found on disk
RemoteUpdateServer revision ahead of local
PackageUpdateNewer package version available
ProviderMismatchProvider expected but not rendered
UnknownFileFile in state but not in current render plans
PolicyDriftLocal config violates workspace policy
NestedTargetDriftMonorepo sub-target missing files
CleanNo drift (used internally)
