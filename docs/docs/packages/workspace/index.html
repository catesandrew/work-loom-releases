Workspace Package

The @workloom/workspace package provides the WorkspaceService for CRUD operations on workspaces and the WorkspaceResolver for resolving a workspace's desired state by resolving package version constraints against the registry.

Usage​

import { WorkspaceService, WorkspaceResolver, MapStore } from '@workloom/workspace'
import { MapStore as RegistryMapStore } from '@workloom/registry'

// Create workspace service with in-memory store
const store = new MapStore()
const service = new WorkspaceService(store)

// Create a workspace
const workspace = await service.createWorkspace({
  orgId: 'org_123',
  name: 'frontend-team',
  binding: {
    workspaceId: '',
    registryUrl: 'https://registry.workloom.dev',
    mode: 'generated-local',
  },
  targets: [
    { id: 'root', path: '.', packages: [], providers: ['claude'] },
  ],
})

// Resolve desired state
const registryStore = new RegistryMapStore()
const resolver = new WorkspaceResolver(registryStore)
const desired = await resolver.resolveDesiredState(workspace)

WorkspaceService​

CRUD service for workspace lifecycle management. Each update increments the workspace revision.

MethodDescription
createWorkspace(options)Create a new workspace (revision starts at 1)
getWorkspace(workspaceId)Fetch a workspace by ID
updateWorkspace(workspaceId, options)Update workspace fields (increments revision)
deleteWorkspace(workspaceId)Delete a workspace

CreateWorkspaceOptions​

FieldTypeDescription
orgIdstringOwning organization
namestringWorkspace display name
bindingBindingRegistry binding configuration
targetsWorkspaceTarget[]?Deployment targets

UpdateWorkspaceOptions​

FieldTypeDescription
namestring?Updated name
bindingBinding?Updated binding
targetsWorkspaceTarget[]?Updated targets

WorkspaceResolver​

Resolves a workspace's declared packages into exact versions by querying the registry and applying semver constraints.

const resolver = new WorkspaceResolver(registryStore)

const state = await resolver.resolveDesiredState(workspace, {
  overrides: [
    { packageName: '@workloom/skills', versionConstraint: '~2.1.0' },
    { packageName: '@workloom/agents', targetId: 'frontend', channel: 'beta' },
  ],
})

ResolvedState​

FieldTypeDescription
workspaceIdstringWorkspace identifier
revisionnumberWorkspace revision at resolution time
packagesResolvedPackage[]Fully resolved packages

ResolvedPackage​

FieldTypeDescription
namestringPackage name
versionstringExact resolved version
sha256stringContent hash
targetIdstringWhich workspace target this belongs to
targetPathstringRelative path from workspace root
providersProvider[]Target providers
channelReleaseChannelRelease channel

WorkspaceOverride​

Overrides allow pinning a package to a specific constraint or channel, optionally scoped to a target.

FieldTypeDescription
packageNamestringPackage to override
targetIdstring?Scope override to a specific target
versionConstraintstring?Override version constraint
channelReleaseChannel?Override channel

Store Interface​

Both MapStore (in-memory) and DrizzleWorkspaceStore (PostgreSQL) implement WorkspaceStore:

interface WorkspaceStore {
  getWorkspace(workspaceId: string): Promise<Workspace | undefined>
  putWorkspace(workspace: Workspace): Promise<void>
  deleteWorkspace(workspaceId: string): Promise<boolean>
  listWorkspaces(options?: ListWorkspacesOptions): Promise<Workspace[]>
}
