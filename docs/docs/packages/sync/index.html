Sync Package

The @workloom/sync package implements the local sync engine that applies render plans to the filesystem. It handles path validation, atomic file writes, orphan cleanup, trust classification, and approval tracking.

Usage​

import { SyncPlanner, LocalApplier } from '@workloom/sync'

// 1. Build a sync plan from render plans
const planner = new SyncPlanner()
const syncPlan = planner.buildPlan(renderPlans, currentState)

// 2. Apply the plan to the filesystem
const applier = new LocalApplier()
const result = await applier.apply(syncPlan, '/path/to/workspace')

console.log(`Written: ${result.written.length}`)
console.log(`Deleted: ${result.deleted.length}`)
console.log(`Skipped: ${result.skipped.length}`)

SyncPlanner​

Produces a SyncPlan by comparing render plans against current state to determine what actions to take.

buildPlan(renderPlans, currentState)​

ParameterTypeDescription
renderPlansRenderPlan[]Rendered output from all providers
currentStateStateFile | nullCurrent .loom/state.json (null on first sync)

Returns a SyncPlan containing:

FieldTypeDescription
renderPlansRenderPlan[]Input plans (preserved for content lookup)
localActionsLocalAction[]Write/delete/skip actions to execute
blockedBlockingDiagnostic[]Actions that cannot proceed without approval

Action Types​

ActionWhen
writeNew file, or content hash changed
deleteFile in current state but not in any render plan
skipFile exists and content hash matches (unchanged)

Path Validation​

The planner validates every output path before planning a write. Invalid paths (traversal attempts, absolute paths, reserved names) are added to blocked diagnostics instead of localActions.

LocalApplier​

Executes a sync plan by writing, deleting, and skipping files atomically.

apply(syncPlan, workspaceRoot)​

Returns an ApplyResult:

interface ApplyResult {
  written: string[]   // Paths successfully written
  deleted: string[]   // Paths successfully deleted
  skipped: string[]   // Paths unchanged (no-op)
  errors: Array<{ path: string; error: string }>  // Failed operations
}

Atomic Writes​

Files are written atomically via AtomicWriter:

  1. Write content to a temporary file in the same directory
  2. Rename (atomic on POSIX) to the target path
  3. Ensures no partial writes on crash or interrupt

Line Ending Normalization​

All written content is normalized to consistent line endings via PathNormalizer.normalizeLineEndings().

Supporting Classes​

PathNormalizer​

Validates and normalizes file paths:

  • Rejects path traversal (../)
  • Rejects absolute paths
  • Rejects reserved OS filenames
  • Normalizes line endings in content
import { PathNormalizer } from '@workloom/sync'

const normalizer = new PathNormalizer()
const error = normalizer.validate('../etc/passwd')
// => { reason: 'Path traversal not allowed' }

AtomicWriter​

Writes files atomically via temp file + rename:

import { AtomicWriter } from '@workloom/sync'

const writer = new AtomicWriter()
await writer.write('/path/to/file.md', content)

TrustClassifier​

Classifies files by trust category to determine approval requirements:

import { TrustClassifier } from '@workloom/sync'

const classifier = new TrustClassifier()
const category = classifier.classify(asset, provider)

ApprovalLedger​

Tracks which files have been approved for write:

import { ApprovalLedger } from '@workloom/sync'

const ledger = new ApprovalLedger()
ledger.approve(path, reason)
const isApproved = ledger.isApproved(path)
